## 🚀 Featured Projects

<details>
<summary><b>⚡ DSA Visualizer — Interactive Algorithm Education Platform</b></summary>
<br/>

> An interactive, real-time visualizer for sorting and searching algorithms built from scratch in React. Designed as both an educational tool and a technical showcase of React architecture, state management, and animation engineering.

| Attribute | Details |
|-----------|---------|
| **Stack** | React 18 · Vite · CSS3 · JavaScript ES6+ |
| **Algorithms** | Bubble · Selection · Merge · Quick · Heap Sort · Binary Search · Insertion Sort |
| **Key Feature** | Pseudocode panel highlights the **exact line executing** at each animation step |
| **Architecture** | Algorithm layer (pure JS) completely decoupled from UI layer (React components) |
| **Animation Engine** | Step-object pattern — algorithms pre-generate all steps, player replays them |
| **Performance** | O(1) step lookup — no re-computation during playback |
| **Security** | No external dependencies, no API keys, fully client-side |
| **Impact** | Live tool for learning DSA with visual + pseudocode + complexity info simultaneously |
| **Repository** | [Source Code](https://github.com/Gunjan-Yadav12/-dsa-visualizer) |
| **Live Demo** | [dsa-visualizer-beta-sepia.vercel.app](https://dsa-visualizer-beta-sepia.vercel.app/) |

**Engineering Highlights:**
- Custom animation engine using `useEffect` + `setInterval` with stale closure prevention via functional state updates
- Separation of concerns — 5 algorithm files with zero React imports, 5 component files with zero algorithm logic
- Reusable `PseudocodePanel` component accepts any algorithm's pseudocode array and active line index
- Time complexity bar dynamically updates on algorithm switch using a `COMPLEXITY` lookup object
- Binary Search includes sorted-array pre-processing and custom target input with conditional rendering

</details>

<summary><b>📊 Credit Card Fraud Detection — End-to-End ML Pipeline</b></summary>
<br/>

> An end-to-end machine learning pipeline for credit card fraud detection on a massively imbalanced real-world dataset. Demonstrates production-grade ML methodology including data engineering, model comparison, and evaluation with business-relevant metrics.

| Attribute | Details |
|-----------|---------|
| **Stack** | Python · Scikit-learn · Pandas · NumPy · Matplotlib · Seaborn · Joblib |
| **Dataset** | 284,807 transactions · 492 fraud cases (0.17% fraud rate) |
| **Scale** | Processed 284K+ records with full feature engineering pipeline |
| **Class Imbalance** | SMOTE oversampling → balanced from 394 to 227,451 fraud samples |
| **Models** | Logistic Regression (AUC: 0.9765) vs Random Forest (AUC: 0.9644) |
| **Best Recall** | Logistic Regression — 90% fraud recall |
| **Best Precision** | Random Forest — 84% fraud precision |
| **Impact** | Demonstrates why accuracy is the wrong metric for imbalanced fraud data |
| **Repository** | [Source Code](https://github.com/Gunjan-Yadav12/credit-card-fraud-detection) |

**Engineering Highlights:**
- Architected modular OOP data pipeline with reusable components and data quality checks
- Applied SMOTE to solve 0.17% class imbalance — naive model gets 99.8% accuracy but catches 0% fraud
- Compared two models using Precision, Recall, F1, AUC-ROC — not just accuracy
- Generated ROC curve comparison and feature importance visualization saved as artifacts
- Saved trained model and scaler with Joblib for production reuse

</details>
